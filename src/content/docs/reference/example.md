Les pages de référence décrivent **comment les choses fonctionnent** (APIs internes, conventions, glossaire), sans scénario utilisateur détaillé.

## Bonnes pratiques

- Structure stable : titres courts, tableaux pour les paramètres ou statuts.
- Liens vers les guides ou runbooks quand une procédure existe.
